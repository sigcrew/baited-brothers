# 어종 판별 QA 데이터

## 실제 어종 사진 후보

`gbif-candidates.json`은 GBIF 발생 기록에서 수집한 공개 라이선스 후보의
출처, 저작자, 라이선스와 검토 상태를 보관한다. 현재 후보는 테스트 정답으로
확정된 데이터가 아니다.

1. `npm run qa:collect-gbif-candidates -- --download`로 후보를 갱신한다.
2. 종 동정, 한 마리 이상 식별 가능 여부, 과도한 편집 여부, 라이선스를 확인한다.
3. 승인한 항목만 `reviewStatus`를 `approved`로 변경한다.
4. 종마다 서로 다른 환경의 승인 사진 5장 이상을 확보한다.
5. 평가 manifest에 승인 사진만 넣고 `npm run qa:fish-recognition`을 실행한다.

현재 수집 결과는 60종 300건으로 종별 후보가 5건씩 있다. 낙지의 부족한
2건은 iNaturalist 공개 라이선스 관찰 기록으로 보충했다. 다운로드 폴더는
용량과 라이선스 재검토 문제로 Git에 포함하지 않는다.

## 거부 데이터

비어류와 저품질 사진은 각각 30장을 구성했다. 비어류는 GBIF 공개 라이선스
관찰 사진이며, 저품질 세트는 앱에서 검수된 실제 어종 사진 5장에 6가지
열화 조건을 적용한 재현 가능한 파생본이다.

- 비어류: 고양이, 개, 토끼, 조류, 불가사리, 게 각 5장
- 저품질: 흔들림, 역광, 야간, 부분 잘림, 작은 피사체

`npm run qa:prepare-rejection-data`를 실행하면 파일과 manifest를 다시 만들 수
있다. 출처와 라이선스, 열화 조건은 `rejection-sources.json`에 기록된다.

외부 AI로 전송되는 사진에는 얼굴, 차량번호, 위치 메타데이터 등 불필요한
개인정보가 없어야 한다.
