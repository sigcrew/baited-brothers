## 배포 전 반드시 완료

### AI 판별

- 후보 300건의 종 동정·품질·라이선스를 사람이 승인
- 비어류·저품질 각 30장은 확보 완료. 전체 세트의 거부율을 아래 평가 명령으로 측정
- `AI_TEST_USER_JWT`를 일시적으로 설정하고 `npm run qa:fish-recognition` 실행
- 합격 기준: Top-3 90% 이상, 비어류·저품질 거부 95% 이상
- 오답 사진은 삭제하지 말고 회귀 테스트 데이터로 유지

사진은 외부 AI 처리자에게 전송되므로, 강화한 AI 함수 배포 전에 운영자의
명시적 승인과 개인정보 처리방침의 제3자 처리 고지가 필요하다.

### 로그인·계정

- Apple 로그인: 실기기에서 최초 로그인, 재로그인, 취소, 이메일 숨기기 확인
- Google 로그인: 개발·프리뷰·프로덕션 빌드 각각 OAuth 리디렉션 확인
- Supabase Auth Redirect URLs에 `baited-brothers://google-auth` 등록 확인
- 이메일 테스트 계정 전체 삭제는 자동 통합 테스트 통과
- Apple 계정은 Edge Function에 Apple 장기 키와 토큰 암호화 키를 설정한 뒤 신규 실계정으로 로그인·토큰 폐기·전체 삭제 확인

### 스토어

- 개인정보 처리방침 GitHub Pages 배포 후 `https://sigcrew.github.io/baited-brothers/privacy/` 공개 확인
- 현재 문의처는 공개 GitHub Issues다. 출시 전 전용 이메일을 사용할 경우 문서와 앱을 함께 교체
- App Store Connect 개인정보 설문에 이메일·사진·정밀 위치·사용자 콘텐츠·진단 항목 신고
- Google Play 데이터 보안 양식과 계정 삭제 안내 URL 작성
- App Store/Play Console 앱 식별자가 `com.sigcrew.baitedbrothers`인지 소유자 확인

### 빌드·서명

- `eas credentials --platform ios`에서 Apple Team과 배포 인증서 확인
- `eas build --platform ios --profile preview`로 실기기 내부 배포 검증
- Apple·Google 로그인 검증 후 production 빌드
- 실제 제출 전 버전·빌드 번호와 스토어 메타데이터 확정

### 도구 체인 보안

- 프로덕션 의존성 감사 결과: critical 0, high 0, moderate 14
- 남은 항목은 Expo SDK 54 도구 체인과 연결되어 있으므로 `npm audit fix --force`로 강제 승격하지 않는다
- SDK 업그레이드 브랜치에서 Expo 호환성 검사, 네이티브 빌드, 로그인·카메라 회귀 테스트와 함께 정리한다
