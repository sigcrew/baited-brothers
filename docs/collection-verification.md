# 수집 인증 룰

> Baited Brothers (낚시당한 녀석들)  
> 원칙: **수집 = 현장에서 카메라로 찍어 어종을 확정한 조과**  
> 위치·카메라는 조작을 **완벽하게 차단**하지 않고, **비용을 올려 집 도배·올클릭을 막는** 현실적 조합이다.

관련 문서: [`positioning-monetization.md`](./positioning-monetization.md) · [`design-direction.md`](./design-direction.md) · [`navigation.md`](./navigation.md) · [`map-view.md`](./map-view.md)

---

## 1. 제품 한 줄

> 수집 성공 조건 = **카메라 실시간 촬영** ∧ **셔터 순간의 GPS** ∧ **(선택) 조업 가능 지역** ∧ **어종 확정(폐쇄형 후보 → 유저 선택)**  
> 사진 없는 수동 추가, 갤러리만으로의 해금, 비전 LLM·모델 **자동 해금**은 허용하지 않는다.

---

## 2. 왜 이 조합인가

| 위협 | 카메라 실시간 | 촬영 시 GPS | 조업존 매칭 | 패턴 이상 탐지 |
|------|---------------|-------------|-------------|----------------|
| 도감에서 올클릭 도배 | 차단 | — | — | — |
| 집에서 갤러리/검색 이미지로 채우기 | 차단에 가까움 | 강하게 차단 | 강하게 차단 | 보조 |
| 옛 사진 EXIF 위장 | 갤러리 금지로 완화 | 셔터 GPS가 진실 | 보조 | 보조 |
| 갯바위에서 모니터/다른 폰 화면 촬영 | 약함 | 약함 (GPS는 정상) | 약함 | 보조 |
| GPS 스푸핑 | 약함 | 약함 | 약함 | 중 (나중) |

**위치는 1번 필터이지 만능 열쇠가 아니다.**  
모니터 사기는 낚시터에서도 가능하므로, 목표는 완전 차단이 아니라 **허위 풀도감의 난이도·비용 상승**이다.

---

## 3. 현실적 조합 (채택)

### 3-1. 카메라 실시간만 해금 (필수 · MVP)

- 도감 해금·시즌 챌린지 인정은 **인앱 카메라로 그 순간 촬영한 사진**만 인정
- 갤러리 선택, 파일 첨부, 화면 녹화 우회는 해금 경로에서 제외
- 어종 확정은 **§4 어종 판별 룰**을 따른다 (AI가 후보를 좁히고 유저가 확정)

**의도:** “목록 탭으로 수집”을 구조적으로 제거

### 3-2. 촬영 시점 위치 필수 (필수 · MVP)

- **갤러리 EXIF GPS를 신뢰하지 않음**
- 셔터를 누르는 순간 앱이 획득한 GPS(또는 동등한 위치 권한 결과)를 기록에 저장
- 위치 권한 거부·측위 실패 시: 저장은 막거나, 저장해도 **도감 해금 불가** (제품에서 하나로 고정)

저장 필드 예시:

| 필드 | 설명 |
|------|------|
| `image_url` | 촬영본 |
| `location_lat` / `location_lng` | 셔터 순간의 좌표 |
| `location_captured_at` | 측위 시각 (셔터와 근접해야 함) |
| `fish_id` | 유저 확정 어종 |
| `capture_method` | `live_camera` 고정 |
| `id_method` | `closed_set_candidates` / `fallback_catalog` |
| `candidate_fish_ids` | 폐쇄형 판별이 제시한 후보 (감사·개선용) |

### 3-3. 조업 가능 지역 대략 매칭 (권장 · MVP 또는 MVP+1)

셔터 GPS가 아래 중 하나에 들어가면 **인증(해금)** , 아니면 **미인증(기록만 / 해금 안 함)** .

| 매칭 방식 | 설명 | 비고 |
|-----------|------|------|
| 해안 N km | 해안선 버퍼 (예: 3~10 km) | 바다·방파제 대략 커버 |
| 등록 낚시터 | 좌대·낚시공원·항구 포인트 DB | 정확도↑, 운영 비용↑ |
| 민물 포인트 | 강·호수·유료터 폴리곤/버퍼 | 민물 유저 오탐 방지에 필요 |

**오탐 완화**

- 존 밖이어도 “기록 저장”은 가능하게 둘지, 아예 막을지 정책 분기
- 추천: **존 밖 = 저장 가능·도감 미해금·UI에 사유 표시** (신규 포인트·내륙 낚시 배려)
- 존 데이터는 처음에 거칠게, 클레임 쌓이면 포인트 추가

### 3-4. 패턴 이상치면 미인증 (나중)

MVP 이후, 어뷰징이 실제로 보일 때 도입.

| 시그널 | 예시 |
|--------|------|
| 단시간 다건 | 같은 좌표에서 10분 내 N종 이상 해금 |
| 순간 이동 | 물리적으로 불가능한 좌표 점프 |
| 연속 동일 구도 | 동일 이미지 해시·극단적 유사도 (화면 촬영 보조) |
| 모의 위치 | mock location / 개발자 옵션 힌트 (플랫폼별) |

이상 시: 기록은 남기되 **해금·시즌·랭킹 미반영**, 필요 시 수동 리뷰.

## 4. 어종 판별 룰

> 판별의 주체는 유저, 시스템은 **선택지를 좁히는 보조**.  
> 후보 생성은 **우리 `fishes` set에 대한 폐쇄형 판별**이다.  
> 비전 LLM 토큰 경로는 기본이 아니며, AI 단독 확정·전체 도감 자유 선택은 채택하지 않는다.

### 4-1. 왜 AI 단독 / 자유 선택이 부족한가

| 방식 | 문제 |
|------|------|
| AI 자동 확정 | 오인식이 곧 잘못된 해금·신뢰 하락 |
| 전체 도감에서 유저 자유 선택 | 실수(유사종) + 고의(희귀종 허위) 모두 쉬움 |
| 비전 LLM에 매번 “이게 뭐니?” | 토큰·콜 비용 선형 증가, 도감에 없는 종·환각 가능 |

### 4-2. 채택 플로우 (후보 확정)

```
사진 촬영 (라이브 카메라)
  → 서버: 우리 fishes set 기준 폐쇄형 판별 (top 2~3)
  → 유저는 그 중에서만 선택
  → 확정 시에만 도감 해금
```

| 단계 | 규칙 |
|------|------|
| 시스템 역할 | `fishes` 테이블 라벨만으로 **상위 2~3종** 노출 |
| 유저 역할 | 후보 중 하나를 탭해 **최종 확정** |
| 금지 | 점수가 높아도 **자동 해금하지 않음** |
| 기본 경로 | 전체 도감 검색은 숨기거나 폴백으로만 |

문서에서 “AI 후보” = **도감 폐쇄형 모델(또는 임베딩 매칭)의 top-k**를 뜻한다. 개방형 챗/비전 LLM 응답이 아니다.

### 4-3. 기술 선택: 폐쇄형 set vs 비전 LLM

후보 2~3마리는 창의적 답이 아니라 **우리 도감 ID 안에서 top-k**면 충분하다.

| 방식 | 과금·비용 | 도감(유한 set) 궁합 | 채택 |
|------|-----------|---------------------|------|
| **비전 LLM** (GPT-4o 등) | 이미지+텍스트 **토큰/콜마다** | 약함 — 없는 종·영문명·환각 | ❌ 기본 경로 금지 |
| **폐쇄형 판별** (`fishes`만) | 추론 1회 (토큰 개념 약함) | **강함** — 항상 `fish_id` | ✅ **기본 경로** |

**채택 파이프라인**

```
촬영 이미지
  → (우리 서버) 임베딩 또는 분류 모델
  → fishes 기준 유사도/확률 top 2~3
  → 유저에게 후보 표시 → 확정
```

| 옵션 | 설명 | 언제 |
|------|------|------|
| **A. 이미지 임베딩 + 도감 대표 사진 매칭** | 각 어종 레퍼런스(`image_url` 등)와 cosine top-k | 데이터 적을 때 시작 |
| **B. 폐쇄형 분류 모델** | 라벨 = `fish_id` N종 | 유저 확정 사진이 쌓이면 |
| **C. 비전 LLM** | 개방형 “이게 뭐니?” | **부트스트랩·최후 폴백만**, 매 건 호출 금지 |

**비용**

- LLM 매 촬영: 유저·시즌 증가에 따라 콜 비용 선형 증가
- 자체/전용 폐쇄형 추론: 콜당 저렴, 후보가 항상 우리 set 안 → 허위 종·환각↓
- 레퍼런스 이미지는 도감 `image_url`·시드 데이터로 부트스트랩 가능

**한 줄:** 후보 생성 = 우리 `fishes` set 폐쇄형 판별. 비전 LLM은 기본 경로가 아니라 최후 폴백/초기 실험용.

### 4-4. 폴백 (후보에 없을 때)

폐쇄형 top-k 신뢰도가 낮거나 유저가 “없음”을 고를 때:

1. “비슷한 종이 없어요” 안내
2. **카테고리/검색 폴백**으로 어종 선택 허용
3. (예외) 비전 LLM은 **운영자 도구·극소수 폴백**으로만 검토 — 일반 유저 매 촬영 경로에 넣지 않음
4. 폴백으로 확정한 기록은 아래 중 하나로 처리 (제품에서 하나로 고정)

| 옵션 | 도감 | 시즌·배지 | 추천 |
|------|------|-----------|------|
| A | 해금 | 정상 반영 | 초보 이탈↓, 어뷰징↑ |
| B | 해금 | **미반영 또는 약하게** | **추천** — 실수는 살리되 고의 이득↓ |
| C | 미해금 (기록만) | 미반영 | 가장 엄격, UX 빡셈 |

**권장: 옵션 B** — 폴백도 도감에는 넣되, 시즌 챌린지·랭킹·희귀 배지는 `id_method = closed_set_candidates` (폐쇄형 top-k) 경로만 인정.
(`ai_candidates`는 동일 의미의 레거시/별칭으로 써도 됨.)

### 4-5. 실수·고의 완화 장치

| 장치 | 내용 |
|------|------|
| 확정 전 비교 카드 | 후보를 나란히 보여 주고, 차이 한 줄 (예: “옆구리 반점”) |
| 유사종 묶음 (선택) | 먼저 “우럭류” 등 그룹 → 세부 종 선택 |
| 해금 후 수정 | 확정 후 **N시간/N일 내 어종 변경 1회** (실수 복구) |
| 확정 재확인 | “이 사진으로 ○○ 맞나요?” 한 번 더 |
| 감사 로그 | `candidate_fish_ids` + 최종 `fish_id` + `id_method` 저장 → 모델·UX 개선 |

### 4-6. 롤아웃

| 단계 | 판별 UX / 기술 |
|------|----------------|
| **MVP (카메라·GPS 먼저)** | 폐쇄형 모델 없이도 동작: 임시 **카테고리 상위 + 검색** (자유 선택 폭 최소화) |
| **MVP+** | **옵션 A** 임베딩+도감 레퍼런스 top-3 + 유저 확정 + 폴백 B |
| **이후** | **옵션 B** 분류 모델 고도화, 비교 카드, 유사종 묶음, 확정 사진으로 재학습 |
| **하지 않음** | 매 촬영마다 비전 LLM 토큰 경로를 기본 후보 생성으로 사용 |

모델이 준비되기 전이라도 **§4-2 UX 골격(좁힌 선택 → 확정)** 을 맞춰 두고, 후보 소스를 나중에 폐쇄형 추론으로 교체한다.

## 5. 수집 상태 (도감)

| 상태 | 조건 | 도감·보상 |
|------|------|-----------|
| 없음 | 해당 어종 인증 기록 0 | 미해금 |
| **인증됨 (수집)** | 라이브 카메라 + 셔터 GPS (+ 조업존) + 어종 확정 | 해금, 완성도 반영 |
| **시즌 인정** | 인증됨 + `id_method = closed_set_candidates` (폴백 B 기준) | 시즌 챌린지·관련 배지 |
| 미인증 기록 | 위치 실패·존 밖·이상 패턴 등 | 저널에는 보일 수 있으나 도감 미반영 (정책에 따름) |

사진 없는 수동 기록 경로는 **두지 않거나**, 두더라도 위 표의 수집/시즌에는 절대 넣지 않는다.

## 6. 유저 플로우

```
[기록하기]
  → 카메라 실행 (갤러리 없음)
  → 셔터 (이 순간 GPS 확보 시도)
  → GPS 실패? → 재시도 / 또는 미해금 안내
  → 조업존 체크 (켜져 있으면)
  → 폐쇄형 판별 top 2~3 제시 (없으면 폴백)
  → 유저가 후보 중 선택 (또는 폴백 선택)
  → “○○ 맞나요?” 확인
  → (선택) 크기·메모
  → 저장
  → 조건 충족 시 도감 해금 연출
```

오프라인: 사진·좌표는 로컬 큐 → 온라인 시 업로드·폐쇄형 후보·존 판정·해금 확정.  
오프라인에서 추론 불가 시: 촬영·GPS만 저장 후, 온라인 되면 후보 선택 화면으로 이어가기.

## 7. MVP 범위 vs 나중

| 항목 | MVP | MVP+ / 나중 |
|------|-----|-------------|
| 라이브 카메라만 해금 | ✅ | — |
| 셔터 순간 GPS 필수 | ✅ | — |
| 조업존 대략 매칭 | ✅ 권장 (해안 버퍼부터) | 포인트 DB 고도화 |
| 어종: 좁힌 선택 → 확정 UX | ✅ (후보 소스는 임시) | 폐쇄형 top-3 (임베딩→분류) |
| 폴백 + 시즌은 폐쇄형 경로만 (옵션 B) | ✅ 정책 확정 | — |
| 비전 LLM 매 촬영 후보 생성 | ❌ 금지 | 최후 폴백·실험만 검토 |
| 해금 후 어종 수정 1회 | ✅ 권장 | — |
| 비교 카드·유사종 묶음 | ❌ 또는 최소 | ✅ |
| 패턴 이상 탐지 | ❌ | ✅ 어뷰징 발생 후 |
| GPS 스푸핑 강탐지 | ❌ | ✅ 필요 시 |
| 이미지 해시·화면 탐지 | ❌ | ✅ 보조 |

## 8. 한계 (문서에 명시)

- **낚시터에서 모니터/다른 기기 화면 촬영**은 GPS·존만으로는 막기 어렵다.
- **GPS 스푸핑**은 클라이언트만으로 완전 차단 어렵다.
- **어종 오선택·폐쇄형 오후보**는 0이 될 수 없다 → 수정 창구·폴백 정책으로 완화.
- 목표는 보안 만능이 아니라, **수집의 의미(현장 증빙)** 를 지키고 올클릭·집 도배·허위 희귀 해금을 막는 것.
- 비전 LLM 토큰 경로를 기본으로 쓰면 비용·환각이 커지므로 **`fishes` 폐쇄형 판별**을 기본으로 둔다.

## 9. UX 카피 초안

- 해금 성공: “현장에서 낚아챈 녀석이 도감에 등록됐어요.”
- 위치 없음: “도감 등록에는 촬영 위치가 필요해요.”
- 조업존 밖: “기록은 남겼어요. 이 위치에서는 도감 해금이 안 돼요.”
- 갤러리 없음: “도감 수집은 그 자리에서 사진 찍을 때만 가능해요.”
- 후보 제시: “이 중 어떤 녀석인가요?”
- 후보 없음: “비슷한 종을 못 찾았어요. 카테고리에서 골라주세요.”
- 확정 확인: “이 사진으로 ○○ 맞나요?”
- 폴백 시즌 제외 (옵션 B): “도감에는 등록됐어요. 시즌 챌린지는 자동 후보 확인 기록만 인정돼요.”
