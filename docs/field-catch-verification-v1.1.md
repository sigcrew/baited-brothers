> Baited Brothers(낚시당한 녀석들)의 조과 판별·기록·수집 보상 정책 제안서

관련 문서: [`offline-policy.md`](./offline-policy.md) · [`collection-verification.md`](./collection-verification.md) · [`fishing-trips.md`](./fishing-trips.md) · [`map-view.md`](./map-view.md)

## 1. 배경과 목표

앱의 핵심 경험은 단순히 사진 속 어종을 맞히는 것이 아니라, 사용자가 낚시 현장에서 잡은 조과를 기록하고 수집 카드를 해금하는 것이다.

따라서 다음 두 개념을 분리한다.

- **AI 어종 판별:** 사진에 보이는 물고기·두족류를 한국 연안 도감 후보 안에서 추천한다.
- **현장 조과 인증:** 실제 낚시 현장에서 앱으로 촬영했다는 조건을 확인한다.

AI 판별 성공만으로 현장 조과가 증명되지는 않는다. v1.1의 목표는 기록 기능을 지나치게 제한하지 않으면서 수집 카드, 최고어, 배지와 같은 보상 요소의 신뢰도를 지키는 것이다.

---

## 2. 현재 배포 버전의 동작

TestFlight `1.0.0 (15)` 기준으로 촬영 위치는 저장되지만 인증 조건으로 사용되지 않는다.

- 서울이나 실내에서도 위치 권한을 허용하면 촬영할 수 있다.
- 촬영 장소와 관계없이 AI 어종 판별을 실행한다.
- 사용자가 후보를 확정하면 조과를 저장할 수 있다.
- 저장된 조과는 도감 해금, 수집 카드와 수집 개수에 반영된다.
- 서울 좌표도 조과에 저장된다.
- 지도는 한국 해안 필터를 적용하므로 내륙 조과가 지도에서 보이지 않을 수 있다.
- 배포 빌드는 인앱 카메라 촬영만 제공하며 사진첩 업로드는 개발 모드에서만 제공한다.

즉, 현재 구현은 **위치가 포함된 사진 판별·기록 기능**이며 아직 **현장 조과 인증 기능**은 아니다.

## 3. 채택할 기록 등급

모든 기록을 차단하는 대신 신뢰 조건에 따라 두 등급으로 나눈다.

| 기능 | 현장 인증 조과 | 일반 기록 |
|------|----------------|-----------|
| AI 어종 판별 | 가능 | 가능 |
| 일지 저장 | 가능 | 가능 |
| 어종별 조과 이력 | 표시 | 표시 |
| 도감·수집 카드 해금 | 반영 | 미반영 |
| 수집 개수 | 반영 | 미반영 |
| 공식 최고어 | 반영 | 미반영 |
| 수집·도전 배지 | 반영 | 미반영 |
| 향후 랭킹 | 반영 | 미반영 |
| UI 표시 | `현장 인증` | `일반 기록` |

### 3.1 현장 인증 조과

앱 카메라로 촬영한 순간의 위치·시각·정확도가 인증 조건을 통과한 기록이다. 수집 및 경쟁 요소에 정상 반영한다.

### 3.2 일반 기록

인증 조건을 충족하지 못했지만 사용자가 보관할 가치가 있는 기록이다. 일지와 어종별 조과 이력에는 남기되, 카드 해금이나 공식 최고어에는 사용하지 않는다.

`표시가 없는 기록`으로 만들지 않고 카드와 상세 화면에 `일반 기록` 상태를 명시한다.

## 4. 현장 인증 기본 조건

v1.1에서 다음 조건을 모두 충족하면 현장 인증 대상으로 본다.

1. 앱의 실시간 카메라로 촬영했다.
2. 셔터 시점에 GPS 좌표를 확보했다.
3. 위치 측정 시각과 촬영 시각이 충분히 가깝다.
4. 위치 정확도가 허용 기준 이내다.
5. 좌표가 한국 해안·항구 또는 허용한 근해 범위에 있다.
6. 서버가 동일한 위치·시각 규칙을 다시 검사한다.
7. 동일 사진의 중복 사용 검사에 통과한다.

초기 권장 기준값은 다음과 같으며 실제 현장 QA 후 조정한다.

| 항목 | 초기 권장값 |
|------|-------------|
| 위치 정확도 | `horizontalAccuracy <= 100m` |
| 측위와 셔터 시각 차이 | 2분 이내 |
| 육지 촬영 | 해안선으로부터 3km 이내 |
| 해상 촬영 | 대한민국 서비스 범위 안의 허용 근해 |
| 중복 사진 | 동일 이미지 해시가 기존 인증 조과에 없을 것 |

위치는 강한 1차 신호이지만 GPS 스푸핑이나 낚시터에서 다른 화면을 촬영하는 행위까지 완전히 방지하지는 못한다. 목표는 절대적인 진위 보장이 아니라 허위 수집의 비용을 높이고 보상 시스템의 신뢰도를 확보하는 것이다.

## 5. 촬영 장소와 업로드 장소의 분리

현장 인증은 **업로드 장소가 아니라 촬영 순간**을 기준으로 한다.

### 5.1 낚시터에서 앱으로 촬영하고 귀가 후 업로드

현장 인증으로 인정해야 한다.

```text
출조지에서 앱 카메라 촬영
  → 촬영 순간 GPS·시각·정확도 확보
  → 사진 해시와 인증 정보를 로컬에 저장
  → 업로드 대기
  → 서울 등 다른 장소에서 네트워크 복구
  → 서버 업로드 및 촬영 당시 정보 검증
  → 현장 인증 조과 확정
```

촬영 후 즉시 업로드하는 것은 인증 조건이 아니다. 촬영 순간에 인증 정보가 생성됐는지가 핵심이다.

### 5.2 기본 카메라로 찍고 사진첩에서 나중에 선택

일반 기록으로 처리한다.

사진 EXIF의 위치와 시각은 수정할 수 있으므로 현장 인증의 단독 근거로 신뢰하지 않는다. AI 판별과 기록 저장은 허용할 수 있지만 수집 카드, 최고어와 배지에는 반영하지 않는다.

## 6. 오프라인 정책

통신이 약한 방파제·갯바위·선상 환경을 고려해 오프라인 촬영을 지원한다.

- 촬영 원본, 최적화 이미지, 위치, 위치 정확도, 촬영 시각과 출조 ID를 로컬 큐에 저장한다.
- 앱 종료나 기기 재부팅 후에도 업로드 대기 상태를 복구한다.
- 네트워크가 복구되면 자동 업로드한다.
- 서버 판정 전까지 상태는 `인증 확인 중`으로 표시한다.
- 서버 검증 성공 시 `현장 인증`으로 전환한다.
- 서버 검증 실패 시 사진을 버리지 않고 사유가 포함된 `일반 기록`으로 전환할지 사용자에게 안내한다.

오프라인 상태에서 AI 판별을 실행할 수 없다면 촬영 증거를 먼저 보존하고, 온라인 복구 후 후보 선택 화면으로 이어간다.

## 7. 사용자 안내 원칙

사용자가 저장한 뒤 보상이 누락됐다고 느끼지 않도록 인증 가능 여부를 가능한 한 촬영 전에 알린다.

### 7.1 촬영 전

현장 인증 가능:

> 현장 인증 준비 완료
> 이 조과는 수집 카드, 최고어와 배지에 반영됩니다.

일반 기록만 가능:

> 현재 위치에서는 일반 기록만 만들 수 있어요.
> 수집 카드, 최고어와 인증 배지에는 반영되지 않습니다.

위치를 확인하고 있는 경우:

> 현장 인증 위치를 확인하고 있어요.

### 7.2 저장 버튼

- 인증 조건 충족: `현장 인증 조과로 저장`
- 인증 조건 미충족: `일반 기록으로 저장`
- 오프라인 대기: `현장 촬영 저장 후 나중에 업로드`

### 7.3 일반 기록 사유

`왜 현장 인증이 안 되나요?`에서 다음 중 정확한 사유를 보여준다.

- 해안·근해 인증 범위 밖
- 위치 권한 없음
- 위치 정확도 부족
- 촬영 시점 위치를 확인하지 못함
- 사진첩에서 가져온 사진
- 동일 사진이 이미 사용됨
- 인증 정보가 손상되었거나 서버 검증에 실패함

### 7.4 귀가 후 업로드 안내

> 현장 촬영이 확인되었습니다.
> 네트워크가 연결되면 자동으로 업로드되며, 다른 장소에서 저장해도 현장 인증은 유지됩니다.

## 8. 데이터 모델 제안

기존 조과 레코드에 다음 필드를 추가하는 방안을 권장한다.

| 필드 | 예시 | 설명 |
|------|------|------|
| `verification_status` | `pending`, `field_verified`, `general_record` | 인증 상태 |
| `verification_reason` | `outside_coastal_zone` | 일반 기록 또는 실패 사유 |
| `captured_at` | ISO 8601 | 실제 셔터 시각 |
| `location_lat` / `location_lng` | 숫자 | 셔터 순간 좌표 |
| `location_accuracy_m` | `24.5` | 수평 위치 정확도 |
| `location_captured_at` | ISO 8601 | 위치 측정 시각 |
| `capture_method` | `live_camera`, `photo_library` | 사진 획득 경로 |
| `image_hash` | SHA-256 또는 지각 해시 | 중복 사진 탐지 |
| `verification_version` | `1` | 적용한 서버 규칙 버전 |
| `verified_at` | ISO 8601 | 서버 검증 완료 시각 |

권장 사유 코드는 다음과 같다.

```text
outside_service_area
outside_coastal_zone
location_permission_denied
location_unavailable
location_accuracy_low
location_timestamp_stale
photo_library_import
duplicate_image
server_verification_failed
legacy_record
```

## 9. 서버 검증 원칙

클라이언트의 `현장 인증` 플래그를 그대로 신뢰하지 않는다.

서버는 다음을 다시 계산한다.

- 사용자 인증 세션
- 촬영 방식
- 좌표의 대한민국 서비스 범위 여부
- 해안·근해 허용 여부
- 위치 정확도
- 촬영 시각과 측위 시각의 차이
- 동일 사용자 및 전체 인증 기록의 이미지 중복 여부
- 필요 시 비정상적인 이동 속도와 단시간 다건 등록

클라이언트와 서버는 같은 `verification_version`의 규칙을 사용해야 한다. 앱에서 가능하다고 표시했지만 서버가 거절하면 정확한 사유와 일반 기록 전환 선택지를 반환한다.

## 10. 수집·통계 반영 규칙

다음 집계는 `verification_status = field_verified`인 조과만 사용한다.

- 도감 최초 해금
- 수집 카드 획득
- 전체 수집 개수와 완성도
- 개인 최고어
- 수집·도전 배지
- 향후 시즌 챌린지와 랭킹

일반 기록은 다음 화면에서 계속 볼 수 있다.

- 조과 일지
- 출조 상세
- 해당 어종의 전체 조과 이력
- 프로필의 일반 활동 통계

화면에서 전체 기록 수와 인증 기록 수를 혼용하지 않는다. 예: `전체 조과 18 · 현장 인증 12`.

---

## 11. 기존 기록 처리

구현 시점 이전 조과에는 새 인증에 필요한 위치 정확도와 촬영 증거가 없을 수 있다.

기본 제안은 다음과 같다.

- 기존 레코드를 `general_record`로 마이그레이션한다.
- 사유는 `legacy_record`로 저장한다.
- 기존 사진과 일지 기록은 유지한다.
- 신규 인증 정책 적용 이후의 카드·최고어·배지는 현장 인증 조과만 집계한다.

단, 기존 사용자의 이미 해금된 카드를 다시 잠글지는 사용자 신뢰에 큰 영향을 주므로 구현 전 별도 제품 결정을 받아야 한다. 대안은 기존 해금을 `레거시 수집`으로 유지하되 인증 마크와 향후 경쟁 집계에서만 제외하는 것이다.

1. 인증 상태·사유·위치 정확도·사진 해시 DB 필드 추가

3. 촬영 시각과 위치 측정 시각 기록
4. 클라이언트 해안·근해 사전 판정
5. 일반 기록 안내와 저장 버튼 분기
6. 오프라인 영구 업로드 큐 구현
7. 서버 인증 함수 및 규칙 버전 구현
8. 수집 카드·최고어·배지 집계 조건 변경
9. 카드·일지·출조 상세에 인증 상태 표시
10. 기존 기록 마이그레이션
11. 중복 이미지 및 이상 패턴 탐지 고도화

## 13. 출시 전 인수 조건

- 서울에서 앱 카메라로 촬영하면 AI 판별은 가능하지만 일반 기록으로 안내된다.
- 일반 기록 저장 전에 카드·최고어·배지 미반영 사실을 명시한다.
- 일반 기록을 저장해도 도감 카드가 새로 해금되지 않는다.
- 해안 인증 범위에서 유효한 GPS로 촬영하면 현장 인증 후보가 된다.
- 낚시터에서 오프라인 촬영한 뒤 서울에서 업로드해도 촬영 당시 조건으로 인증된다.
- 사진첩에서 가져온 사진은 위치 EXIF가 있어도 일반 기록이 된다.
- 서버가 인증 조건을 재검증한다.
- 동일 사진을 반복 등록해 현장 인증 보상을 중복 획득할 수 없다.
- 지도·일지·출조 상세에서 인증 상태가 일관되게 표시된다.
- 수집 개수, 최고어와 배지는 현장 인증 조과만 집계한다.

---

## 14. 구현 전 확정이 필요한 항목

- 해안선 허용 거리: 3km 유지 또는 조정
- 선상 낚시를 위한 근해 서비스 범위
- 민물 낚시를 v1.1 인증 범위에 포함할지 여부
- 위치 정확도 기준과 측위 제한 시간
- 서버 검증 실패 시 자동 일반 기록 전환 또는 사용자 확인
- 기존 해금 카드의 유지·잠금·레거시 표시 정책
- 일반 기록도 도감의 어종별 이력에 표시할지 여부
- 사진첩 가져오기를 정식 기능으로 제공할지 여부

## 15. 최종 제품 원칙

> **수집 보상은 촬영 당시의 현장 증거를 기준으로 하고, 업로드 장소와 시점에는 종속되지 않는다.**

- 현장 인증 조과만 도감 해금, 카드, 최고어와 배지에 반영한다.
- 인증하지 못한 사진도 일반 기록으로 보관할 수 있다.
- 사용자가 촬영·저장 전에 어떤 등급으로 기록되는지 알 수 있어야 한다.
- 오프라인 출조와 귀가 후 업로드는 정상적인 현장 인증 흐름으로 지원한다.
- 완벽한 부정 방지보다 명확한 규칙, 서버 재검증과 일관된 UX를 우선한다.
