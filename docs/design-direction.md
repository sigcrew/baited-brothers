# 디자인 방향

> Baited Brothers (낚시당한 녀석들)  
> 기준: **수집형 도감** + 라이트 게이미피케이션  
> 톤: 프리미엄 취미 저널 **70** : 수집 재미 **30**

---

## 1. 한 줄 톤

> 바닷바람 나는 피싱 저널 — 담백하고, 조금 멋있고, 자랑하고 싶은 카드

게이미피케이션이 들어가도 UI가 게임처럼 보이면 브랜드가 싸진다.  
목표는 **도감을 열고 싶어지는 물건**이지, 미니게임이 아니다.

| 축 | 방향 | 피할 것 |
|----|------|---------|
| 분위기 | 바다·새벽·출조 노트 | 네온, 보라 그라데이션, 다크모드 기본 |
| 장난기 | 이름·카피·뱃지에만 | 이모지 범벅, 둥근 알약 CTA 도배 |
| 게임감 | 진행률·완성도만 또렷하게 | XP바, 레벨, 가챠 연출 |
| 정보 | 도감은 읽기 쉽게 | 신문형 다단, 카드 지옥 |

---

## 2. 비주얼 방향

### 2-1. 브랜드 히어로

- 첫 화면에서 **「낚시당한 녀석들」**이 제일 크게 보여야 한다.
- 기능 문구·통계·프로모가 브랜드를 덮지 않는다.
- 브랜드를 뺀 첫 화면이 다른 앱처럼 보이면 브랜딩이 약한 것이다.

### 2-2. 실물 앵커

- 물고기 사진, 바다·방파제·출조 분위기 등 **실제 맥락**이 주인공.
- 추상 그라데이션·장식만으로 히어로를 대체하지 않는다.
- 랜딩/프로모 성격 화면에서는 히어로 이미지를 풀블리드로 둔다. (인셋 카드형 히어로 지양)

### 2-3. 컬러

| 역할 | 방향 |
|------|------|
| Base | 딥 네이비, 포말(거품) 화이트 |
| Accent | 청록 / seafoam |
| 보조 | 모래·로프 느낌의 따뜻한 뉴트럴 (포인트만) |

**쓰지 말 것 (AI 클리셰)**

- 퍼플 온 화이트 / 퍼플→인디고 그라데이션
- 크림 배경 + 세리프 디스플레이 + 테라코타 액센트 조합
- 기본 다크모드, 글로우, 멀티 레이어 섀도
- 라운드 풀 필(pill) CTA 남발

CSS 변수로 토큰을 정의하고, 화면마다 즉흥 hex를 늘리지 않는다.

### 2-4. 타이포그래피

- **제목**: 개성 있는 한글 디스플레이 (낚시·바다 감성과 맞는 것)
- **본문**: 읽기 좋은 고딕
- Inter / Roboto / Arial / system 기본 스택만으로 끝내지 않는다.
- 헤드라인이 브랜드명보다 커지지 않게 위계를 잡는다.

### 2-5. 도감 카드

- 수집 UX의 **주인공 컴포넌트**.
- 기본 카드는 담백하게. 테두리·프레임·스킨은 시즌 패스 **꾸미기**로 남긴다.
- 히어로에는 카드형 UI를 쓰지 않는다.
- 상호작용이 없는 곳에 카드(테두리·그림자·라운드 박스)를 남발하지 않는다.

### 2-6. 모션

의도적인 모션 **2~3개**만 둔다.

| 예시 | 용도 |
|------|------|
| 도감 해금 | 새 어종 등록 시 짧은 등장 |
| 진행률 채움 | 완성도 % 변화 |
| 조과 카드 등장 | 공유 전 미리보기 |

파티클 폭죽, 과도한 바운스, 상시 글로우는 쓰지 않는다.  
모션은 존재감·위계용이지 장식용 노이즈가 아니다.

## 3. 화면별 감도

| 화면 | 감도 | 하지 말 것 |
|------|------|------------|
| **홈** | 출조 일정(계획→완료). 브랜드·수집 진행률은 가볍게 | 계정/로그아웃, 대시보드 통계 나열 |
| **수집** | 도감 \| 뱃지 \| 카드. 도감·앨범이 주인공 | 필터·뱃지·칩 잔치, 별도 「내 기록」탭 |
| **프로필** | 계정·로그아웃·(이후) 설정 | 수집·일정 콘텐츠를 넣지 않음 |
| **기록** | (탭 아님) 기록하기 CTA → 카메라 플로우 | 게임 HUD, XP/레벨 UI |
| **시즌 / 뱃지** | 여기서만 약간의 축제감 (한정 프레임, 뱃지) | 상점형 가챠 연출 |
| **결제 / 패스** | 담백하고 신뢰감 있게 | “현질 샵” 느낌 |

### 홈 첫 뷰포트 예산

보통 이 구성만:

1. 브랜드  
2. 헤드라인 하나  
3. 짧은 보조 문장 하나  
4. CTA 그룹  
5. 지배적인 이미지(또는 도감 진행 시그널)

통계 스트립, 스케줄, 프로모 칩, 떠다니는 배지는 첫 화면에 올리지 않는다.  
히어로 미디어 위에 분리된 라벨·스티커·콜아웃을 올리지 않는다.

## 4. 섹션 구성 원칙

- **한 섹션 = 한 목적** — 헤드라인 하나, 보조 문장 짧게.
- **카드 기본값 = 없음** — 사용자 조작 컨테이너일 때만 카드.
- **잡동사니 금지** — 필 클러스터, 아이콘 로우, 박스형 프로모, 메타데이터 줄 남발 금지.
- **게이미피케이션은 시스템에** — 화면 장식에 XP/에너지/일일퀘스트 UI를 깔지 않음.  
  (상세: `docs/positioning-monetization.md` §5)

## 6. 완성 기준 (검수 체크)

- [ ] 스크린샷만 봐도 “낚시 도감 앱”으로 보이는가
- [ ] 뱃지·진행률을 빼도 여전히 멋있는가 (빼면 허전해지면 게임 과다)
- [ ] 시즌 스킨을 얹었을 때 “더 자랑하고 싶다”가 느껴지는가
- [ ] 브랜드를 가리면 다른 일반 앱처럼 보이지 않는가
- [ ] 보라/크림테라코타/다크글로우 클리셰에 빠지지 않았는가
